### Component Summary: Mercury+ST1 Base Board (ME-ST1-W-R1)

#### Key Specifications:
- **Voltage Ratings**: 
  - Supply Voltage: 12V DC (nominal)
  - I/O Voltage: Configurable (depends on the Mercury module)
- **Current Ratings**: 
  - Maximum current for VCC_MAIN: 2A
- **Power Consumption**: 
  - Power budget and consumption depend on the connected Mercury module and peripherals.
- **Operating Temperature Range**: 
  - -20°C to +75°C
- **Package Type**: 
  - Form factor: 120mm x 100mm
- **Special Features**: 
  - Supports USB 3.0 host and device connections
  - Equipped with multiple I/O connectors including HDMI, Mini DisplayPort, and MIPI interfaces
  - Low-jitter clock generator
  - MicroSD card slot
  - Designed for rapid prototyping and educational purposes
- **Moisture Sensitive Level (MSL)**: 
  - Not specified in the provided text.

#### Description:
The **Mercury+ST1 Base Board** is a versatile development platform designed for use with the Enclustra Mercury family of FPGA and SoC modules. It provides a comprehensive set of I/O connectors and features that facilitate rapid prototyping and development of FPGA-based systems. The board is particularly suited for applications in image processing, allowing users to integrate various peripherals and interfaces seamlessly.

#### Typical Applications:
- **Rapid Prototyping**: The Mercury+ST1 base board allows developers to quickly test and validate their designs without the need for custom hardware.
- **Educational Purposes**: It serves as an excellent tool for teaching and learning about FPGA technology and embedded systems.
- **Image Processing**: The board is specifically designed to support image processing applications, making it suitable for projects involving cameras and video processing.
- **Production Flash Programming**: It can be used for programming Mercury modules in production environments.

This summary encapsulates the essential characteristics and applications of the Mercury+ST1 Base Board, providing a clear understanding of its capabilities and intended use cases.
